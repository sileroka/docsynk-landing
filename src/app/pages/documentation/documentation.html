DocSynk Documentation

Complete guide to using DocSynk for logistics document management

Overview

DocSynk is a specialized document management system built for international logistics companies. It handles bills of lading, customs declarations, shipping manifests, and compliance documentation across multiple jurisdictions.

Key Concepts

  • Shipments: Core organizational unit, groups all documents for a single logistics movement
  • Documents: Individual files (PDFs, images, EDI files) attached to shipments
  • Compliance Rules: Automated checks against country-specific requirements
  • Workflows: Approval chains for document processing

Quick Start

1. Create Your First Shipment

Navigate to Shipments â†’ New Shipment. Enter:

  • Shipment reference (Bill of Lading number)
  • Origin and destination countries
  • Mode of transport

2. Upload Documents

Drag and drop documents or use the upload button. Supported formats:

  • PDF (bills of lading, invoices)
  • Images (JPG, PNG for packing lists)
  • EDI files (X12, EDIFACT)
  • Excel/CSV (manifest data)

3. Review and Share

Once processed, share with partners using role-based permissions.

Document Upload

Upload Methods

Web Interface

Drag and drop files directly onto the shipment page or use the upload button.

Email Gateway

Forward documents to your unique email address:

shipments@your-company.docsynk.cloud

Subject line format: [Shipment-ID] Document Type

API Upload

Use our REST API for programmatic uploads:

POST /api/v1/shipments/{shipmentId}/documents
Content-Type: multipart/form-data

{
  "file": binary,
  "type": "bill_of_lading",
  "metadata": {
    "origin": "system_name"
  }
}

Bulk Operations

Upload multiple documents at once:

  • ZIP files are automatically extracted
  • Documents are matched to shipments by reference numbers
  • Duplicates are detected and flagged

OCR Processing

Automatic Extraction

DocSynk automatically extracts key data from logistics documents:

Bill of Lading Fields

  • B/L Number
  • Shipper/Consignee information
  • Port of loading/discharge
  • Container numbers
  • Commodity description
  • Weight and volume

Commercial Invoice Fields

  • Invoice number and date
  • Buyer/Seller details
  • HS codes
  • Values and currencies
  • Incoterms

Confidence Scores

Each extracted field includes a confidence score:

  • Green (95%+): High confidence, auto-approved
  • Orange (80-94%): Review recommended
  • Red (<80%): Manual verification required

Compliance Checks

Automated Validations

US Customs (CBP)

  • ISF 10+2 filing requirements
  • ACE document standards
  • Partner Government Agency requirements

European Union

  • ENS (Entry Summary Declaration)
  • EXS (Exit Summary Declaration)
  • EORI validation
  • Brexit-specific requirements for UK

China Customs

  • CCD (China Customs Declaration)
  • CIQ inspection requirements
  • Advance manifest rules

Custom Rules Engine

Create custom compliance rules for your specific needs:

IF destination_country = "USA" 
   AND commodity_type = "TEXTILES"
THEN REQUIRE document_type = "TEXTILE_VISA"

Workflows

Standard Workflows

Document Approval

  1. Operations team uploads
  2. Compliance review
  3. Manager approval (if value > $50k)
  4. Release to partners

Amendment Process

  1. Request amendment
  2. Original issuer approval
  3. Update all linked documents
  4. Notify all parties

API Overview

RESTful API for programmatic access to DocSynk.

Authentication

curl -H "Authorization: Bearer YOUR_API_KEY" \
     https://app.docsynk.cloud/api/v1/shipments

Common Endpoints

  • GET /api/v1/shipments - List shipments
  • POST /api/v1/shipments - Create shipment
  • POST /api/v1/documents/upload - Upload document
  • GET /api/v1/compliance/check - Run compliance check

Webhooks

Configure webhooks to receive real-time notifications about shipment events.

Available Events

  • shipment.created - New shipment created
  • document.uploaded - Document added to shipment
  • compliance.passed - Compliance check passed
  • compliance.failed - Compliance check failed

EDI Support

DocSynk supports industry-standard EDI formats for seamless integration with trading partners.

Supported Standards

  • X12 (ANSI ASC X12)
  • EDIFACT (UN/EDIFACT)
  • XML-based standards (GS1, RosettaNet)

User Management

Role Types

Internal Roles

  • Admin: Full system access
  • Manager: Approve high-value shipments
  • Operations: Create and manage shipments
  • Viewer: Read-only access

External Roles

  • Customer: View their shipments
  • Carrier: Upload POD, update status
  • Customs Broker: Submit declarations
  • Partner: Limited document access

Single Sign-On (SSO)

DocSynk supports SSO via:

  • SAML 2.0
  • OAuth 2.0
  • Active Directory/LDAP
